JS中需要注意的地方(更新)
2019-10-21
阅读量:

JS 的延迟加载

  1. befer:表示脚本可以延迟到文档完全被解析和显示后在执行,difer 属性只适用于外部引用脚本文件,在 XHTML 中,要把 befer 属性设置为 brder=”beder”
  2. async:与 defer 属性相似,async 只适用于外部脚本,并告诉浏览器立即下载此文件,于 defer 不同的是,有 async 的属性脚本并不能保证按照他们的先后顺序执行.所以确保两者互不依赖,async 的目的是不让页面等待两个脚本下载和执行,从而异步加载页面的其他内容,因此建议异步脚本 加载不要在加载期间修改 dom,异步脚本一定会在 load 事件前执行.
  3. 支持异步脚本的浏览器有火狐 3.6 Safari5 和 Chrome

基本概念

  1. 语法

ECMA 的语法大量借鉴了 C 及其其他类 C 语言,比如 Java 等

  1. 区分大小写
    ECMA 中的一切(变量,函数名,操作符)都要区分大小写,这就意味着,变量名 Abc 和变量名 abc 分别表示两个不同的变量,并且要采用驼峰命名法
  2. 不要使用关键字保留字等作为变量名或函数名

数据类型

  • typeof:检测数据类型
  • undefined:建议不要使用这个名命名,在以前版本中并没有规定这个值是保留字.而在第三版中则区分了空对象指针未初始化的变量
